## BLOCKING Findings

### [B1] Section 5 prose contradicts section 5's own reported data

**Location:** `book/chapters/ch02/03-poisson.tex` lines 83-108

**Finding:** The figure caption (lines 73-80) and the bullet list (lines 88-91) both correctly report that the MSE model produces **45 negative-rate predictions** with a minimum of **-0.351** on the 201-point grid. This is a materially bad failure: 45 out of 201 predictions (22%) are non-physical. Yet the surrounding prose makes three claims that directly contradict this data:

1. Line 83: "On this benign synthetic dataset both models fit the conditional mean reasonably well."
   This is false given the data in hand. A model that generates 22% non-physical (negative) predictions on the evaluation grid has not "fit reasonably well" in any sense that justifies the word "well". The MSE model is demonstrably broken for count data here and now, not only as a hypothetical.

2. Lines 94-95: "The MSE model could predict any negative number IF the training data were less benign."
   The capital "IF" framing is factually wrong: the MSE model DID predict negative numbers on this dataset. The condition is not hypothetical; it materialized. This sentence should be removed or rewritten to acknowledge the realized failure.

3. Lines 107-108: "Use the matching head because the matching head encodes the structural constraint, even when on a benign dataset the constraint would not have been violated anyway."
   "would not have been violated anyway" directly contradicts 45 negative predictions on a 201-point grid. The constraint WAS violated. The word "anyway" inverts the pedagogical message: the chapter's own experiment is the evidence for the argument, but this sentence dismisses that evidence.

**The contradiction is sharp and pedagogically damaging.** The chapter correctly calls the experiment honest ("Findings, honestly"), yet the framing language sanitizes a real failure into a hypothetical. A reader who reads the bullet list and then reads the conclusion will notice the contradiction and lose trust in the author's reliability.

**Auditors:** spec-auditor (plan §8 lists "internal consistency" as highest-priority check), quality-auditor (honest-empiricism commitment from master spec §3 and §14), math-auditor (numbers confirmed internally consistent with themselves but the surrounding prose mischaracterizes them).

**Suggested fix:** Rewrite the "Findings, honestly" block:
- Replace "both models fit the conditional mean reasonably well" with an honest characterization: the Poisson model tracks the true rate throughout; the MSE model tracks the mean in the bulk but produces negative predictions in 45 of 201 evaluated points (22%), which are non-physical for a count model.
- Replace the "could predict ... IF the training data were less benign" sentence: the condition already occurred. The MSE model did predict negative rates here.
- Replace the closing sentence with the correct version: use the matching head because the structural constraint is violated on this dataset, not merely a risk on worse datasets.

### [S6] The NLL formula in section 5 drops log(y!) for optimization but the loss value also drops it

**Location:** `book/chapters/ch02/03-poisson.tex` lines 18-22; `src/scratchnn/neural_net.py` lines 229-231

The text says: "The log y! term does not depend on z and drops from the gradient. For the loss value during optimization we omit it as well (it shifts the loss by a y-dependent constant but does not affect the optimum)." The code confirms: `value` returns `math.exp(z) - z * y` with no `log(y!)` term.

The omission is mathematically correct for optimization. However, the prose notes it is valid to omit for the loss value "during optimization." The chapter then uses the loss value directly in the comparative test Poisson NLL evaluation (notebook cell-6, function `poisson_nll_at`). In the notebook, `poisson_nll_at` uses `rate - y * math.log(rate)`, which also omits `log(y!)`. The comparative NLL values (MSE Poisson NLL vs Poisson head Poisson NLL) are therefore not proper NLL values; they are NLL minus a constant that depends on y. For a comparison between two models on the same test set this is valid (the constant cancels), but the prose does not state this. If a reader tries to use these NLL values to compare against a benchmark from the literature, they will get a misleading number.

The heteroscedastic test NLL (0.745 vs 0.509) also omits the 0.5*log(2*pi) constant (acknowledged in the text for the loss but not for the evaluation). Both omissions are consistent and standard; the concern is that the comparative-evaluation prose does not note this limitation explicitly.

**Auditors:** math-auditor, quality-auditor.

**Suggested fix:** Add a parenthetical to the Poisson comparative NLL discussion noting that the NLL values shown omit the `log(y!)` term (a per-example constant), so they represent relative rather than absolute Poisson NLL; the comparison is valid but the absolute values should not be compared to literature benchmarks.
