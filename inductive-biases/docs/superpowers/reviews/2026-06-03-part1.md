## SUBSTANTIVE Findings

### [S1] Chapter 2 notebook: narrative cells state seeds/epochs that contradict the executed code

- **Location:** `notebooks/ch02-output-heads.ipynb` cell `f565d70f` (header markdown)
  and cell `fef6b53b` (Results Summary markdown), versus executed cells `f7d7dd4d`,
  `85ea9b78`, `bdbfd3ac` (Poisson experiment).
- **Finding:** The header cell and the Results Summary cell both state the Poisson
  experiment used "data seed=42, model seed=42, 200 epochs." The actually executed
  code sets `DATA_SEED = 0`, `MODEL_SEED = 0`, and trains with `epochs=80`. The
  drafted prose (`book/chapters/ch02/03-poisson.tex:71`) says "(data seed~0, model
  seed~0, 80 epochs)", which matches the executed code, not the notebook's own
  prose. So the prose is correct against the run, but the notebook is internally
  inconsistent: its narrative claims 42/42/200 while its code does 0/0/80. This
  directly undermines Part-I design decision (C), "seeds are fixed and recorded,"
  because the recorded seeds disagree with the run that produced the numbers the
  prose cites. A reader who trusts the header and re-runs with seed 42 / 200 epochs
  will not reproduce 0.1524 / 0.1387.
- **Lenses:** math-auditor (provenance of the cited numbers), spec-auditor
  (decision C: notebook is the seed-locked source of truth).
- **Suggested fix:** Make the notebook's header and Results Summary cells state
  `data seed=0, model seed=0, 80 epochs` to match the executed code (and the prose).
  This is a notebook-prose edit only; no number changes.

### [S2] Both notebooks' "Results Summary" cells say "see cell output" instead of recording the literal numbers

- **Location:** `notebooks/ch02-output-heads.ipynb` cell `fef6b53b`; same pattern

### [S3] The residual symbol is never reconciled across chapters: Chapter 1 writes `\hat{y} - y`, Chapter 2's keystone theorem writes `\hat{p} - y`

- **Location:** `book/chapters/ch01/01-foundations-to-limits.tex:22,24,26,46`
  (`\hat{y} - y` for MSE) versus `book/chapters/ch02/02-canonical-link.tex:34-35,48,
  62-63,71-72` (`\hat{p} - y`, with MSE re-expressed as "`\hat{p} - y` with
  `\hat{p} = z`").
- **Finding:** This is the notation seam the part-level review was asked to
  scrutinize, and it is real but mild. Chapter 1 introduces the regression residual
  as `\hat{y} - y` (eq:residual) and the classification gradient as `p - y`
  (eq:py-binary) and `\vp - \vy` (eq:py-softmax). Chapter 2's canonical-link theorem
  unifies everything under `\hat{p} - y`, and it does explicitly fold MSE in:
  "MSE (identity plus Gaussian) is another: `\partial L/\partial z = z - y`, which
  is `\hat{p} - y` with `\hat{p} = z`" (02-canonical-link.tex:71-72). That sentence
  does the reconciliation for the `z`/`\hat{p}` identity. What is NOT explicitly
  reconciled is the Chapter 1 symbol `\hat{y}`: a careful reader sees three glyphs
  for the model's prediction across the part (`\hat{y}` in Ch1 MSE, `p` in Ch1
  classification, `\hat{p}` in Ch2's general theorem) and is left to infer that
  `\hat{y} = \hat{p} = z` for the Gaussian/identity case. The inference is easy and
  the math is correct, but the part never says it in one place. Because the
  canonical-link theorem is explicitly sold as the keystone that explains "no
  coincidence," the one symbol it does not tie back is the very first one the reader
  met (`\hat{y}`).
- **Lenses:** math-auditor (symbol consistency), quality-auditor (the keystone
  result should close the loop on its own opening notation).
- **Suggested fix:** Add half a sentence in `sec:canonical-link` (after line 72)
  noting that Chapter 1's regression residual `\hat{y} - y` is the same instance,
  with `\hat{y} = \hat{p} = z` under the identity link. Alternatively, when
  `sec:three-pairings` recaps "Identity plus Gaussian plus MSE" (02-canonical-link.tex:79-85),
  write the gradient there as `\hat{p} - y = \hat{y} - y` once, explicitly bridging
  the two glyphs. One line closes the seam.

### Verdict: the `p - y` thread's three appearances -- MATHEMATICALLY CONSISTENT AND PAID OFF

1. **Introduced, Chapter 1.** `eq:py-binary` (`\partial L/\partial z = p - y`,
   03-output-heads.tex:27) and `eq:py-softmax` (`p_i - \mathbf{1}[i=y]`, i.e.
   `\vp - \vy`, 03-output-heads.tex:84). The seed is planted in
   `sec:nn-as-function` too (the `\hat{y} - y` residual, eq:residual, with prose
   "this pattern ... recurs throughout the chapter ... a theorem about exponential
   families and canonical links ... Chapter 2 proves it," lines 26-30).
2. **Generalized, Chapter 2.** `thm:canonical-link` with `eq:expfam-grad`
   (`-T(y) + A'(z) = \hat{p} - y`, 02-canonical-link.tex:46-49), proved from the
   exponential-family identity `A'(\eta) = \E[T(y)]`. The proof is terse but valid;
   it explicitly back-references `\Cref{sec:logistic}` and `\Cref{sec:softmax}` as
   the two instances it generalizes (lines 7-13), discharging the Chapter 1 promise.
3. **Instanced again, Chapter 2.** Poisson: `\partial L/\partial z = e^z - y =
   \lambda - y = \hat{p} - y`, called "A third instance of `\Cref{thm:canonical-link}`,
   with no new derivation needed" (03-poisson.tex:31-34).

The three forms are consistent: binary `p - y`, multiclass `\vp - \vy`, Poisson
`\lambda - y`, all the single statement `\hat{p} - y` (expected sufficient statistic
minus observed). The only loose thread is notational, not mathematical: the Chapter 1
`\hat{y}` glyph for the regression case is never explicitly equated to `\hat{p}` (see
[S3]). The narrative genuinely pays off the Chapter 1 "not an accident" promise.
