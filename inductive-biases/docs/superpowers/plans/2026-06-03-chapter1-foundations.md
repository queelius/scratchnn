## 1. Goal

Draft Chapter 1, "Foundations: Function Approximation and the Multilayer Perceptron," by porting and elevating the existing booklet chapter into the bookwright structure. The chapter develops the network as a parameterized function approximator, proves the single linear unit cannot represent XOR, recovers the MLP and backpropagation as local composition, introduces the two output heads (`SigmoidBCE`, `SoftmaxCrossEntropy`), and lands the book's thesis (two axes of inductive bias) as a preview that Chapter 2 completes. Every code listing is real `scratchnn`; every number is regenerated by the paired notebook.

## 2. Architecture Summary

- Book class, `pdflatex` plus `biber` (per `book/Makefile`).
- `book/book.tex` inputs `book/parts/part1.tex`, which (after this plan) inputs the Chapter 1 wrapper. The wrapper inputs per-section-group files.
- Sections are auto-numbered (`secnumdepth=2`, already set in `preamble.tex`); cross-references use `\Cref`; labels are kebab-case with type prefixes.
- Code is typeset with `listings` (added to the preamble in Task 1; pure pdflatex, no shell-escape).
- The chapter teaches from `../../src/scratchnn/neural_net.py`. Listings are pulled from source, lightly trimmed (init boilerplate elided, hand-derived `backward` kept in full).

## 3. Tech Stack

- LaTeX: book class, amsmath/amssymb, biblatex (biber, authoryear), hyperref, listings, microtype, alex.sty notation.
- Notebook: Python + uv + Jupyter (`bookwright.config.yaml`). `scratchnn` installed editable (`uv pip install -e ../..`). `sklearn` for the UCI digits dataset (`sklearn.datasets.load_digits`); `matplotlib` for the XOR boundary figure. NumPy permitted in the notebook; the `scratchnn` core stays pure Python.
